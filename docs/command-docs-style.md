# 命令文档编写规范

本文档规定 cloud139 项目 README 及帮助文档中命令示例的编写标准，确保格式统一、易于理解。

## 1. 语法格式规范

### 1.1 参数符号约定

遵循 POSIX/GNU 标准：

| 符号 | 含义 | 示例 |
|------|------|------|
| `<参数>` | 必填参数（用户必须提供） | `<本地路径>` |
| `[参数]` | 可选参数（用户可省略） | `[远程目录]` |
| `[参数...]` | 可选且可重复（0 个或多个） | `[文件...]` |
| `<参数...>` | 必填且可重复（1 个或多个） | `<源路径...>` |
| `[-s <值>]` | 短选项带值 | `[-s <存储类型>]` |
| `[--long <值>]` | 长选项带值 | `[--cloud-id <云盘ID>]` |
| `\|` | 多选一 | `--storage-type <personal_new\|family\|group>` |

**禁止使用的符号：**
- 禁止用 `(参数)` 表示可选参数
- 禁止用 `{参数}` 除非表示枚举值集合
- 禁止中英文符号混用（如用中文方括号［］）

### 1.2 命令语法行格式

```bash
cloud139 <COMMAND> [选项...] [位置参数...]
```

具体要求：
- **命令名** 放在最前面
- **全局选项**（如 `-v`）在各命令语法中省略，在"全局选项"章节统一说明
- **选项** 放在位置参数之前
- **必填位置参数** 用 `<>`，**可选位置参数** 用 `[]`
- 位置参数名使用**描述性中文**，如 `<远程路径>`、`<本地路径>`
- 语法行中的参数名应**描述清晰**，避免过于抽象的命名（如 `<类型>`、`<ID>`），应使用 `<存储类型>`、`<云盘ID>` 等具体名称
- 多个同类参数用空格分隔，可重复的加 `...`

**正确示例：**

```bash
cloud139 login -t <TOKEN> [-s <存储类型>] [-c <云盘ID>]
cloud139 ls [路径] [-p <页码>] [-s <数量>] [-o <输出文件>]
cloud139 upload <本地路径> [远程目录] [-f]
cloud139 mv <源路径...> <目标路径> [-f]
```

**错误示例：**

```bash
# ❌ 圆括号表示可选
cloud139 download <远程路径> (本地路径)

# ❌ 必填参数用方括号
cloud139 upload [本地路径] [远程目录]

# ❌ 中英文混用且命名不统一
cloud139 sync <SRC> <DEST>
```

### 1.3 选项与参数的书写顺序

在语法行中，按以下顺序排列：

1. 必填选项（如 `-t <TOKEN>`）
2. 常用可选选项
3. 不常用可选选项
4. 必填位置参数
5. 可选位置参数

**示例：**

```bash
cloud139 login -t <TOKEN> [-s <存储类型>] [-c <云盘ID>]
cloud139 mv <源路径...> <目标路径> [-f]
```

## 2. 位置参数命名规范

### 2.1 命名风格

统一使用**中文描述性名词**：

| 类型 | 命名示例 | 说明 |
|------|----------|------|
| 文件路径 | `<文件路径>` / `[文件路径]` | 泛指文件或目录 |
| 本地路径 | `<本地路径>` | 强调在本地文件系统 |
| 远程路径 | `<远程路径>` | 强调在云盘 |
| 目录 | `[目录]` / `<目标目录>` | 强调是目录而非文件 |
| 名称 | `<新名称>` | 纯文件名，不含路径 |
| Token/ID | `<TOKEN>` / `<ID>` | 标识符类可保留英文大写 |

**统一性要求：**
- 同一个概念在全文必须保持相同名称
  - ✅ `upload <本地路径> [远程目录]` + `download <远程路径> [本地路径]`
  - ❌ `upload <本地文件路径> [目录]` + `download <远程文件路径> [保存路径]`

### 2.2 大小写约定

- 中文参数名：不加引号，如 `<远程路径>`
- 英文参数名：全大写，如 `<TOKEN>`、`<ID>`
- 选项值：小写英文，如 `[-s <存储类型>]`

## 3. 命令介绍结构规范

每个命令的文档必须包含以下部分（按顺序）：

### 3.1 标题与简介

```markdown
### <命令名>

<一句话描述命令的作用。>
```

要求：
- 简介不超过 30 字
- 以动词开头（登录、列出、上传、删除等）

### 3.2 语法行

```markdown
```bash
cloud139 <命令> <语法>
```
```

要求：
- 必须提供语法行
- 语法行中只出现本命令特有的选项和参数
- 全局选项不在此处重复

### 3.3 参数说明表

```markdown
**参数说明：**

| 参数 | 简写 | 必填 | 说明 |
|------|------|------|------|
| ... | ... | ... | ... |
```

表格列要求：
- 必须有**参数**、**必填**、**说明**三列
- 如果该命令没有简写选项，**简写**列可以省略
- **必填**列用"是/否"表示
- 默认值在**说明**列中注明，格式为"默认：X"

**示例：**

```markdown
| 参数 | 简写 | 必填 | 说明 |
|------|------|------|------|
| 路径 | - | 是 | 要删除的文件或目录路径 |
| --yes | -y | 是 | 确认删除 |
| --force | -f | 否 | 强制继续，默认：false |
```

### 3.4 使用示例

```markdown
**示例：**

```bash
# <场景注释>
cloud139 <命令> <具体参数>

# <场景注释>
cloud139 <命令> <具体参数>
```
```

**要求：**
- **每个示例命令的上方一行必须提供注释**，说明该示例的场景或目的
- **不同示例命令之间必须用空行分隔**
- 第一个示例应为**最简单用法**（最少参数）
- 后续示例展示**常见选项组合**或**典型场景**
- 注释以 `#` 开头，后接一个空格，再跟描述性文字
- 示例中的参数值应**具体且真实**，避免 `<>` 或 `[]`
- **示例中使用的选项参数，优先使用缩写形式**（如用 `-p` 而非 `--page`，用 `-f` 而非 `--force`）

**正确示例：**

```bash
# 基础用法：列出根目录
cloud139 ls /

# 列出指定目录
cloud139 ls /myfolder

# 分页查看，每页50条，查看第2页（使用缩写参数）
cloud139 ls / -p 2 -s 50

# 强制上传，使用缩写参数
cloud139 upload ./file.txt / -f
```

**错误示例：**

```bash
# ❌ 缺少注释
cloud139 ls /

# ❌ 注释在同一行右侧
cloud139 ls / # 列出根目录

# ❌ 缺少空行分隔
cloud139 ls /
cloud139 ls /myfolder -p 2

# ❌ 注释在命令下方
cloud139 ls /
# 列出根目录

# ❌ 示例中使用长参数而非缩写（应使用 -f）
cloud139 upload ./file.txt / --force
```

## 4. 快速开始区规范

README 顶部的"快速开始"中的注释示例应与详细命令参考保持**严格一致**。

**格式：**

```bash
# <场景描述>
cloud139 <命令> <具体示例>
```

要求：
- **注释应描述示例的场景和命令用途**，而非命令语法
- 具体示例展示**最常用场景**
- 同样需要**注释在命令上方**、**示例之间空行分隔**
- 如果示例中包含选项参数，**优先使用缩写形式**

**正确示例：**

```bash
# 列出根目录文件
cloud139 ls /

# 列出指定目录文件
cloud139 ls /myfolder

# 上传文件到根目录
cloud139 upload ./file.txt /

# 上传文件到指定目录
cloud139 upload ./photo.jpg /backup/

# 下载文件到当前目录
cloud139 download /file.txt

# 下载文件到指定路径
cloud139 download /file.txt ./downloads/myfile.txt
```

**错误示例：**

```bash
# ❌ 注释是命令语法而非场景描述
# ls [路径]
cloud139 ls /

# ❌ 圆括号
# download [远程文件路径] (本地保存路径)

# ❌ 与详情页不一致
# rm [远程文件路径] -y    # 详情页是 <路径>

# ❌ 在注释示例中展开不常用选项
# sync [本地路径] [cloud:远程路径] -r

# ❌ 缺少空行分隔
cloud139 ls /
cloud139 ls /myfolder
```

## 5. 全局选项规范

全局选项在单独的"全局选项"章节中统一说明，**不在各命令语法行中重复**。

```markdown
## 全局选项

| 参数 | 简写 | 默认值 | 说明 |
|------|------|--------|------|
| --verbose | -v | info | 日志级别 (trace, debug, info, warn, error) |
```

## 6. 一致性检查清单

在更新 README 或编写新命令文档时，对照以下清单检查：

- [ ] 所有必填参数使用 `<>`，所有可选参数使用 `[]`
- [ ] 位置参数命名在全文中保持一致（如都用"路径"或"目录"，不混用）
- [ ] 快速开始区的注释描述场景和命令用途，而非命令语法
- [ ] 每个命令都有：简介、语法行、参数表、示例
- [ ] 参数表的"必填"列无遗漏
- [ ] 默认值在说明列中明确标注
- [ ] 不使用圆括号 `()` 表示可选参数
- [ ] 示例中使用具体值，而非 `<参数>` 占位符
- [ ] 示例命令上方必须有场景注释
- [ ] 不同示例之间有空行分隔
- [ ] 示例中的选项参数优先使用缩写形式
- [ ] 语法行中的参数命名描述清晰，避免 `<类型>`、`<ID>` 等模糊命名
